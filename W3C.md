# 01 W3C(World Wide Web Consortium)
- 웹의 장기적인 성장을 보장하기 위해 웹 기술의 작동 방식과 규칙을 표준화를 목표로 하는 조직들이 모여 협력하는 연합체
- 웹 기술에 대한 공식적인 표준 사양을 제정하고, 브라우저 제조사와 관련 산업 주체들이 해당 사양을 따르도록 웹 기술의 공통 사용 설명서를 만드는 역할을 한다.
- ex) HTML, CSS, XML이 어떻게 작동해야 하며 웹 서버와 브라우저가 어떤 형식으로 데이터를 주고받고 데이터의 의미를 어떻게 해석해야 하는지를 표준으로 정의

### 왜 표준화를 하는가?
- 표준화가 없으면 브라우저 마다 데이터의 의미를 다르게 해석하고 작동 방식도 달라  웹 페이지가  브라우저 마다  다르게 보일 수 있고 새로운 기술을 도입할 때 마다 명확한 기준이 없다면 호환성 문제 발생

# 02 AI 에이전트의 등장
### 배경 및 문제의식
- 기존 인터넷 인프라는 주로 인간의 상호작용을 중심으로 설계되어, AI 에이전트가 상호작용하기에는 데이터 고립, 비효율적인 인터페이스, 높은 협업 비용 등의 문제가 존재한다.

### 왜 기본 인터넷 인프라로 AI 에이전트가 상호작용 하지 못하는가?
- 기존 웹 데이터는 사람을 위해 표현됨(텍스트, 이미지 PDF등), 사람은 문맥, 경험으로 의미를 해석, AI 에이전트의 경우 명시적 구조, 기계가 해석 가능한 의미가 코드에 없으면 해석 불가
- UI는 사람의 상호작용을 전제로 설계됨, AI 에이전트는 빨간색 표시 -> 경고 와 같이 직관적으로 이해가능 반면 에이전트의 경우 의미를 명시적으로 알 수 없음
- 에이전트는 암묵적 신호를 이해하지 못하기 때문에 상태와 의도가 공유되어야 함, UI는 의도를 표현하지 않기 때문에 에이전트 협업간의 정보 부재

# 03 문제해결을 위한 움직임
인간 중심 웹에서 AI 에이전트가 직접 이해하고 상호작용 가능한 에이전트 웹으로의 확장

### W3C AI Agent Protocol
- AI 에이전트 간 상호작용을 위한 공통 규칙을 정의하고, 에이전트가 자신의 능력, 상태, 작업 의도를 명시적으로 표현하고 교환할 수 있는 방법을 제안
- 기존의 UI 중심 상호작용에서 프로토콜 중심 상호작용으로의 전환, AI 에이전트가 UI를 거치지 않고 직접 기능을 발견하고 협업할 수 있는 구조를 목표
- 에이전트 간 작업의 분산과 조율을 가능하게 함으로써 협업 비용을 줄이고, 상태와 의도를 공유하여 중복 작업과 충돌감소
- CNP(Contract Net Protocol)와 같은 분산 협업 메커니즘을 웹 환경에 적용하려는 시도
